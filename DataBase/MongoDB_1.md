# MongoDB - Intro

## 1. NoSQL DBMS
* 질의 명령어가 SQL이 아니다.
* 정보의 형식을 미리 정하지 않는다.

* (Not Only SQL) 전통적인 관계형 데이터베이스 보다 덜 제한적인 일관성 모델을 제공하는 DB이다.
* 특징
    * 확장성, 유연성, 고성능, 고기능성

## 2. MongoDB
* 현대적 설계: 분산 컴퓨팅을 하는 것이 기본값
* 활용
    1. 프로젝트 언어를 JS로 통일시키고자 할 때 
    * 프론트: React, 백엔드: node.js, DB: mongoDB
    2. 파일럿 프로젝트
    3. 비 정형화된 정보를 전산화 - 로그 데이터
    4. 안정성보다 높은 성능이 필요한 경우
